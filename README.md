# Project: lain-day

## 项目目标 (Project Goal)

本作的核心目标是，在 `Gemini CLI` 环境下，合作完成一部具有中篇小说体量的、可交互的文本游戏。并在创作过程中，不断探索和确定其独特的游戏类型。

## 核心理念 (Core Concepts)

本项目旨在通过一个非线性的、深度分支的叙事体验，探索身份认同、现实与虚拟、记忆与“可能性”等核心主题。玩家的选择，甚至“不选择”，都将深刻地影响故事的走向和主角Lain对世界的认知。

## 主要特色 (Main Features)

*   本项目在完成前，没有核心特色。
*   它唯一的过程性特色是：故事可以不断地“分叉”(branch)出新的可能性，也可以在未来将这些可能性“合并”(merge)到一起。

## 设计与开发哲学 (Design & Development Philosophy)

*   **记录迭代困难 (Documenting Iterative Challenges):** 本 `README` 及项目文件也将用于记录在开发迭代过程中遇到的核心困难与解决方案。
*   **版本即设想 (Version as Vision):** 将不同阶段的想法和灵感直接固化为故事分支或文件，让项目本身成为设计思路的备忘录。

## 当前进度 (Current Progress)

*   **序章 (Prologue):** 故事网络和所有分支结局的文本设计已全部完成。
*   **状态系统:** 已确立了用于衔接后续章节的核心“态度”状态。

## 下一步计划 (Next Steps)

1.  **故事创作:** 开始构思和编写第一章的内容。
2.  **引擎开发:** 编写一个简单的游戏引擎，用于解析和运行已设计的序章故事文件。
